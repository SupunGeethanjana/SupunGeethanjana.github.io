# Supun Geethanjana - Portfolio Website

A premium, high-performance personal portfolio website built for an Associate Tech Lead. This project showcases professional experience, skills, testimonials, and blog insights using modern web technologies.

![Portfolio Preview](./public/Supun.png)

## üöÄ Technologies Used

- **[Vite](https://vitejs.dev/)**: Next Experience Frontend Tooling.
- **[React](https://reactjs.org/)**: JavaScript library for building user interfaces.
- **[Tailwind CSS](https://tailwindcss.com/)**: A utility-first CSS framework for rapid UI development.
- **[Framer Motion](https://www.framer.com/motion/)**: Production-ready motion library for React animations.
- **[React Icons](https://react-icons.github.io/react-icons/)**: SVG reactions icons.
- **[GitHub Pages](https://pages.github.com/)**: Static site hosting.

## ‚ú® Features

- **Responsive Design**: Fully adaptable layout for mobile, tablet, and desktop.
- **Dynamic Animations**: Smooth scroll reveals and hover effects.
- **Modern UI/UX**: Custom color palette (Dark Slate & Sky Blue) and glassmorphism elements.
- **Downloadable Resume**: Direct link to download the PDF resume.
- **Testimonial Showcase**: Grid layout for peer and client endorsements.
- **Blog Integration**: Links to external content on Medium, LinkedIn, and Dev.to.

## üõ†Ô∏è Installation & Setup

1.  **Clone the repository**
    ```bash
    git clone https://github.com/SupunGeethanjana/SupunGeethanjana.github.io.git
    cd SupunGeethanjana.github.io
    ```

2.  **Install dependencies**
    ```bash
    npm install
    ```

3.  **Run local development server**
    ```bash
    npm run dev
    ```
    Open [http://localhost:5173](http://localhost:5173) to view it in the browser.

## üì¶ Deployment

This project is configured for deployment on **GitHub Pages**.

1.  **Build and Deploy**
    The `deploy` script automatically builds the project before pushing to the `gh-pages` branch.
    ```bash
    npm run deploy
    ```

2.  **Configuration**
    Ensure your GitHub repository settings under **Pages** are set to serve from the `gh-pages` branch.
